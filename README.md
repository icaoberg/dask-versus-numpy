A benchmarking notebook comparing [Dask](https://dask.org/) and [NumPy](https://numpy.org/) for matrix multiplication and large dataset I/O.

## What it does

### Matrix multiplication benchmark
Measures wall-clock time for square matrix multiplication across sizes N = 101–2500, comparing:
- `numpy.matmul` — in-memory computation
- `dask.array.matmul` — chunked, lazy computation (forced via `.compute()`)

Both use the same random seed (`6174`) for reproducibility.

### Large dataset I/O benchmark
Generates a synthetic TSV dataset (fake PII — SSN, name, phone, company, credit card) and compares read performance between `pandas.read_csv` and `dask.dataframe.read_csv` at two scales:
- 25,000 rows (`fake_dataset.tsv`)
- ~1.5 million rows (`data.tsv`, 175 MB)

## Usage

Open `Dask vs Numpy.ipynb` in Jupyter and run all cells in order. The `data.zip` file must be present in the working directory for the large dataset section.

## Notes

- The fake dataset contains synthetic PII generated by the `faker` library — no real personal data is used.
- Dask's read time advantage for large files reflects lazy evaluation; actual computation occurs on `.compute()`.
